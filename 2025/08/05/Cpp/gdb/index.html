本文主要记录gdb的入门方法, 及在Linux下使用gdb调试C++多线程的方法

quick start

首先给出一段C++程序

将其编译为可调试的程序,需要加上 -g 选项

随后使用 gdb main 开始调试

list
可以查看源代码

r (run)
可以运行

break / b
(line / function name) 在某一行打一个断点

  • 比如 break 4 会在第四行打一个断点, 运行时gdb会停在第四行, break main会在main所在的行打一个断点
  • info b可以查看打的所有的断点

n (next)
执行到下一行

p (print)
print value, 可以是表达式
如 p j, p k, 甚至是 p (j * 2 + (k * k)) 这样的表达式

shell + ${shell command}
执行shell命令

set logging on
可以将输出信息保存到一个文件中

watch x
打一个watch point在x上,如果x发生变化则会通知

调试core文件

由于Linux会对用户的shell做出限制, 默认不会生成core文件
可以通过ulimit -a 查看当前shell的限制

gdb error core

t + 线程号 切换到线程
